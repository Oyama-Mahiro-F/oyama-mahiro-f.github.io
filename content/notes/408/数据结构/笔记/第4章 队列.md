### （一）队列的基本概念

#### 一、队列的定义

##### 1. 核心定义与特性

队列也是一种特殊的**线性表**，其操作受限：只允许在表的一端插入，在另一端删除。

- **核心原则**：**FIFO (First In, First Out，先进先出)**。最先入队的元素必须最先出队，就像排队买票，先来的人先服务。

##### 2. 基本术语

- **队头 (Front)**：允许进行删除操作的一端，用队头指针 `front` 指示队头元素。
- **队尾 (Rear)**：允许进行插入操作的一端，用队尾指针 `rear` 指示队尾元素（或队尾的下一个位置，取决于实现）。
- **空队列**：不包含任何数据元素的队列。
- **入队 (Enqueue)**：在队尾添加新元素的操作。
- **出队 (Dequeue)**：将队头元素删除并取出的操作。

##### 3. 常见的实际应用

队列的 FIFO 特性使其在处理具有"排队"、"缓冲"、"层序"性质的问题时不可或缺：

- **树的层次遍历**：从根节点开始，按层从上到下、从左到右依次访问节点。队列保存当前层的所有节点。
- **图的广度优先搜索 (BFS)**：从起始点开始，逐层扩散访问，队列保存待访问的邻接顶点。
- **操作系统进程调度**：就绪队列按 FCFS（先来先服务）原则分配 CPU；多级反馈队列综合优先级和时间片。
- **打印机缓冲池 / 磁盘 I/O 请求**：多任务共享资源的排队机制，按请求顺序依次处理。
- **页面替换算法 (FIFO)**：操作系统内存管理中，淘汰最早进入内存的页面。
- **消息队列**：生产者-消费者模型中解耦的中间件，异步传递消息。

#### 二、队列的抽象数据类型定义

抽象数据类型（Abstract Data Type, ADT）包含三个核心部分：**数据对象**、**数据关系**和**基本操作**。

##### 1. 数据对象与数据关系

- **数据对象**：$D = \{a_1, a_2, \dots, a_n\}$，其中 $n \ge 0$。每个 $a_i$ 属于同一种数据类型（ElemType）。
- **数据关系**：$R = \{ <a_{i-1}, a_i> | a_{i-1}, a_i \in D, i=2,\dots,n \}$。
  - 这是一个严格的线性关系。
  - 约定 $a_1$ 为**队头**（Front），$a_n$ 为**队尾**（Rear）。

##### 2. 基本操作 (Operations)

在考察和实际编码中，队列的接口通常被定义为以下几个核心原子操作：

- **`InitQueue(&Q)`**
  - **操作前提**：无。
  - **操作结果**：构造一个空队列 `Q`，分配必要的内存空间。
- **`DestroyQueue(&Q)`**
  - **操作前提**：队列 `Q` 已存在。
  - **操作结果**：销毁队列 `Q`，释放其占用的所有内存资源。
- **`QueueEmpty(Q)`**
  - **操作前提**：队列 `Q` 已存在。
  - **操作结果**：若队列为空则返回 `true`，否则返回 `false`。
- **`EnQueue(&Q, x)`**
  - **操作前提**：队列 `Q` 已存在且未满。
  - **操作结果**：将元素 `x` 插入队尾，使其成为新的队尾元素。
- **`DeQueue(&Q, &x)`**
  - **操作前提**：队列 `Q` 已存在且**非空**。
  - **操作结果**：删除队头元素，并用引用参数 `x` 返回其值。
- **`GetHead(Q, &x)`**
  - **操作前提**：队列 `Q` 已存在且**非空**。
  - **操作结果**：读取当前的队头元素，用参数 `x` 返回其值。**注意：此操作不会改变队列的状态（元素不出队）**。

### （二）顺序队列

#### 一、基本顺序队列

用**一段物理地址连续的存储空间**（数组）来存储队列元素。

- **核心结构定义 (C/C++)：**

```c
#define MAXSIZE 50          // 定义队列中元素的最大个数

typedef struct {
    int data[MAXSIZE];      // 存放队列元素的静态数组
    int front;              // 队头指针
    int rear;               // 队尾指针
} SqQueue;
```

- **指针含义**：`front` 指向队头元素，`rear` 指向队尾元素的**下一个空位**（即待插入位置）。
- **初始化**：`front = rear = 0`，表示空队列。

##### 致命问题：假溢出 (False Overflow)

下面用一组连续的入队/出队操作演示"假溢出"的形成过程。

**【步骤 1】初始空队列**

```
初始 (front=rear=0)：
┌───┬───┬───┬───┬───┬───┐
│   │   │   │   │   │   │
└───┴───┴───┴───┴───┴───┘
  ↑
front=0, rear=0
```

**【步骤 2】连续入队 A、B、C**

```
入队 A → rear=1：         入队 B → rear=2：         入队 C → rear=3：
┌───┬───┬───┬───┬───┬───┐  ┌───┬───┬───┬───┬───┬───┐  ┌───┬───┬───┬───┬───┬───┐
│ A │   │   │   │   │   │  │ A │ B │   │   │   │   │  │ A │ B │ C │   │   │   │
└───┴───┴───┴───┴───┴───┘  └───┴───┴───┴───┴───┴───┘  └───┴───┴───┴───┴───┴───┘
  ↑       ↑                  ↑           ↑              ↑               ↑
front=0  rear=1            front=0     rear=2         front=0          rear=3
```

*此时 `front=0, rear=3`，队列中有 3 个元素 (A, B, C)。*

**【步骤 3】连续出队 A、B**

```
出队 A → front=1：        出队 B → front=2：
┌───┬───┬───┬───┬───┬───┐  ┌───┬───┬───┬───┬───┬───┐
│   │ B │ C │   │   │   │  │   │   │ C │   │   │   │
└───┴───┴───┴───┴───┴───┘  └───┴───┴───┴───┴───┴───┘
      ↑       ↑                      ↑   ↑
    front=1  rear=3               front=2|
                                       rear=3 
```

*此时 `front=2, rear=3`，队列中只剩 1 个元素 (C)。`front` 前面有 2 个空位 (下标 0 和 1)。*

**【步骤 4】尝试入队 D → 假溢出！**

```
入队 D：
┌───┬───┬───┬───┬───┬───┐
│   │   │ C │ D │   │   │      rear 到达下标 4
└───┴───┴───┴───┴───┴───┘
              ↑       ↑
            front=2  rear=4

继续入队 E → rear=5 → 再入队 F 时 rear 已经 >= MAXSIZE
→ 虽然下标 0、1 空闲，但 rear 无法向前移动，报"队满"
→ 这就是 "假溢出"：数组有空间但指针已越界
```

```
假溢出示意：
┌───┬───┬───┬───┬───┬───┐
│   │   │ C │ D │ E │ F │    front=2, rear=6 → 报满！
└───┴───┴───┴───┴───┴───┘
  ↑   ↑               ↑
 空闲 空闲           rear=MAXSIZE (越界)
```

> **假溢出的本质**：`front` 之前腾出的空间无法被 `rear` 利用，因为 `rear` 只会单调递增。

#### 二、循环队列（顺序队列的核心考点）

##### 1. 核心思想

将数组**逻辑上视为环形**——当 `rear`（或 `front`）到达数组末尾时，通过**取模运算**将其绕回到数组头部：

$$\text{rear} = (\text{rear} + 1) \bmod \text{MAXSIZE}$$
$$\text{front} = (\text{front} + 1) \bmod \text{MAXSIZE}$$

##### 2. 循环队列完整操作演示

以下以 `MAXSIZE = 6` 为例，展示从空队列开始的完整操作过程。

**【步骤 1】初始空队列**

```
   ┌───┬───┬───┬───┬───┬───┐
   │   │   │   │   │   │   │
   └───┴───┴───┴───┴───┴───┘
     ↑
  front=0, rear=0
```

**【步骤 2】入队 A、B、C、D**

```
入队 A (rear=1)：          入队 B (rear=2)：
┌───┬───┬───┬───┬───┬───┐  ┌───┬───┬───┬───┬───┬───┐
│ A │   │   │   │   │   │  │ A │ B │   │   │   │   │
└───┴───┴───┴───┴───┴───┘  └───┴───┴───┴───┴───┴───┘
  ↑       ↑                    ↑       ↑
front=0  rear=1              front=0  rear=2

入队 C (rear=3)：          入队 D (rear=4)：
┌───┬───┬───┬───┬───┬───┐  ┌───┬───┬───┬───┬───┬───┐
│ A │ B │ C │   │   │   │  │ A │ B │ C │ D │   │   │
└───┴───┴───┴───┴───┴───┘  └───┴───┴───┴───┴───┴───┘
  ↑           ↑                ↑           ↑
front=0     rear=3            front=0     rear=4
```

*此时 `front=0, rear=4`，队列中有 4 个元素 (A, B, C, D)。*

**【步骤 3】出队 A、B**

```
出队 A (front=1)：         出队 B (front=2)：
┌───┬───┬───┬───┬───┬───┐  ┌───┬───┬───┬───┬───┬───┐
│   │ B │ C │ D │   │   │  │   │   │ C │ D │   │   │
└───┴───┴───┴───┴───┴───┘  └───┴───┴───┴───┴───┴───┘
      ↑           ↑                  ↑       ↑
    front=1     rear=4             front=2  rear=4
```

*此时 `front=2, rear=4`，队列中有 2 个元素 (C, D)。下标 0、1 空闲。*

**【步骤 4】继续入队 E、F、G（rear 绕回）**

```
入队 E (rear=5)：          入队 F：rear=(5+1)%6=0！   入队 G：rear=(0+1)%6=1
┌───┬───┬───┬───┬───┬───┐  ┌───┬───┬───┬───┬───┬───┐  ┌───┬───┬───┬───┬───┬───┐
│   │   │ C │ D │ E │   │  │ F │   │ C │ D │ E │   │  │ F │ G │ C │ D │ E │   │
└───┴───┴───┴───┴───┴───┘  └───┴───┴───┴───┴───┴───┘  └───┴───┴───┴───┴───┴───┘
      ↑               ↑      ↑           ↑               ↑       ↑
    front=2        rear=5   rear=0     front=2         rear=1  front=2
                              (绕回来了！)
```

*入队 F 时，`rear` 从 5 → (5+1)%6 = 0，成功绕回到数组头部！这就是循环队列解决假溢出的关键。*

**【步骤 5】判满：牺牲一个单元**

```
此时状态：
┌───┬───┬───┬───┬───┬───┐
│ F │ G │ C │ D │ E │   │     front=2, rear=1
└───┴───┴───┴───┴───┴───┘
      ↑       ↑
    rear=1  front=2
    下标1 已用，下标5 为空（牺牲的单元）

判满检查：(rear+1) % 6 = (1+1) % 6 = 2 == front → 队满！
```

##### 3. ⚠️ 核心难点：判空 vs 判满

循环队列中，`front == rear` 既是空队条件也是满队条件。**必须区分！** 三种经典方案：

| 方案 | 判空条件 | 判满条件 | 元素个数 | 特点 |
|:---|:---|:---|:---|:---|
| **牺牲一个单元** | `Q.front == Q.rear` | `(Q.rear + 1) % MAXSIZE == Q.front` | `(Q.rear - Q.front + MAXSIZE) % MAXSIZE` | **408 最常考，默认方案** |
| **增设 size 变量** | `Q.size == 0` | `Q.size == MAXSIZE` | `Q.size` | 直观，需在结构体中增加 `size` 字段 |
| **增设 tag 标记** | `Q.front == Q.rear && Q.tag == 0` | `Q.front == Q.rear && Q.tag == 1` | 取决于 tag | `tag=0` 上次是出队，`tag=1` 上次是入队 |

> 考研默认采用**牺牲一个单元**的方案。最大容纳元素数 = `MAXSIZE - 1`。

##### 4. 循环队列核心操作（牺牲一个单元方案，408 默认）

> `front = rear = 0` 为初始值，`rear` 指向下一个空位。

###### ① 初始化 (InitQueue)

```c++
void InitQueue(SqQueue &Q) {
    Q.front = Q.rear = 0;       // 初始都指向下标 0，表示空队列
}
```

| 项目 | 内容 |
|:---|:---|
| 初始值 | `Q.front = Q.rear = 0` |

###### ② 判空 (QueueEmpty)

```c++
bool QueueEmpty(SqQueue Q) {
    return Q.front == Q.rear;   // 头尾指针重合即为空
}
```

| 项目 | 内容 |
|:---|:---|
| 判空条件 | `Q.front == Q.rear` |

###### ③ 判满 (QueueFull)

```c++
bool QueueFull(SqQueue Q) {
    return (Q.rear + 1) % MAXSIZE == Q.front;
}
```

| 项目 | 内容 |
|:---|:---|
| 判满条件 | `(Q.rear + 1) % MAXSIZE == Q.front` |
| 原理 | rear 再前进一步就追上 front，预留一个空单元 |

###### ④ 入队 (EnQueue)

```c++
bool EnQueue(SqQueue &Q, int x) {
    // ① 先判满
    if ((Q.rear + 1) % MAXSIZE == Q.front)
        return false;

    // ② 存入 rear 指向的空位
    Q.data[Q.rear] = x;

    // ③ rear 环形后移
    Q.rear = (Q.rear + 1) % MAXSIZE;
    return true;
}
```

| 项目 | 内容 |
|:---|:---|
| 时间复杂度 | 严格 $O(1)$ |
| 核心步骤 | `Q.data[Q.rear] = x; Q.rear = (Q.rear + 1) % MAXSIZE`（先存再移） |
| 判满 | `(Q.rear + 1) % MAXSIZE == Q.front` 时返回 false |

###### ⑤ 出队 (DeQueue)

```c++
bool DeQueue(SqQueue &Q, int &x) {
    // ① 先判空
    if (Q.front == Q.rear)
        return false;

    // ② 取出队头元素
    x = Q.data[Q.front];

    // ③ front 环形后移
    Q.front = (Q.front + 1) % MAXSIZE;
    return true;
}
```

| 项目 | 内容 |
|:---|:---|
| 时间复杂度 | 严格 $O(1)$ |
| 核心步骤 | `x = Q.data[Q.front]; Q.front = (Q.front + 1) % MAXSIZE`（先取再移） |
| 判空 | `Q.front == Q.rear` 时返回 false |

###### ⑥ 元素个数 (QueueLength)

```c++
int QueueLength(SqQueue Q) {
    return (Q.rear - Q.front + MAXSIZE) % MAXSIZE;
    // +MAXSIZE 处理 rear 绕回后小于 front 的情况
}
```

| 项目 | 内容 |
|:---|:---|
| 公式 | `(Q.rear - Q.front + MAXSIZE) % MAXSIZE` |

###### ⑦ 读队头 (GetHead)

```c++
bool GetHead(SqQueue Q, int &x) {
    if (Q.front == Q.rear)
        return false;           // 队空
    x = Q.data[Q.front];        // 直接读，不出队
    return true;
}
```

| 项目 | 内容 |
|:---|:---|
| 时间复杂度 | $O(1)$ |
| 注意 | 只读取不修改 `front`，元素仍在队中 |

##### 5. front 和 rear 的初始化方式

与栈游标 `top` 类似，循环队列中 `front` 和 `rear` 的初始化方式也直接决定了判空、判满、入队、出队的代码写法。**考研中必须根据题目给定的初始条件来推导对应操作**。

以下以 `MAXSIZE = 5` 为例（牺牲一个单元判满）。

###### 方式一：`front = rear = 0`（rear 指向下一个空位，★ 408 默认）

```
初始 (front=rear=0)：      入队 A (rear=1)：         入队 B (rear=2)：
┌───┬───┬───┬───┬───┐    ┌───┬───┬───┬───┬───┐    ┌───┬───┬───┬───┬───┐
│   │   │   │   │   │    │ A │   │   │   │   │    │ A │ B │   │   │   │
└───┴───┴───┴───┴───┘    └───┴───┴───┴───┴───┘    └───┴───┴───┴───┴───┘
  ↑                            ↑                        ↑
front,rear=0                 front=0,rear=1           front=0,rear=2

出队 A (front=1)：          再入队 C (rear=3)：       再入队 D, E → 队满：
┌───┬───┬───┬───┬───┐    ┌───┬───┬───┬───┬───┐    ┌───┬───┬───┬───┬───┐
│   │ B │   │   │   │    │   │ B │ C │   │   │    │ E │ B │ C │ D │   │
└───┴───┴───┴───┴───┘    └───┴───┴───┴───┴───┘    └───┴───┴───┴───┴───┘
      ↑       ↑                ↑       ↑            ↑           ↑
    front=1  rear=2           front=1  rear=3      rear=0     front=1
                                                    (rear+1)%5 = 1 == front → 满！
```

| 操作 | 伪代码 | 说明 |
|:---|:---|:---|
| 初始化 | `Q.front = Q.rear = 0` | 都指向下标 0 |
| 判空 | `Q.front == Q.rear` | 两指针重合 |
| 判满 | `(Q.rear + 1) % MAXSIZE == Q.front` | rear 再进一步就追上 front |
| 入队 | `Q.data[Q.rear] = x; Q.rear = (Q.rear + 1) % MAXSIZE` | 先存入，rear 再后移 |
| 出队 | `x = Q.data[Q.front]; Q.front = (Q.front + 1) % MAXSIZE` | 先取出，front 再后移 |
| 读队头 | `x = Q.data[Q.front]` | front 直接指向队头 |
| 元素个数 | `(Q.rear - Q.front + MAXSIZE) % MAXSIZE` | +MAXSIZE 处理绕回 |

###### 方式二：`front = 0, rear = -1`（rear 指向当前队尾元素）

```
初始 (front=0,rear=-1)：  入队 A(rear→0)：         入队 B(rear→1)：
┌───┬───┬───┬───┬───┐    ┌───┬───┬───┬───┬───┐    ┌───┬───┬───┬───┬───┐
│   │   │   │   │   │    │ A │   │   │   │   │    │ A │ B │   │   │   │
└───┴───┴───┴───┴───┘    └───┴───┴───┴───┴───┘    └───┴───┴───┴───┴───┘
  ↑    ↑                      ↑     ↑                    ↑     ↑
front=0 rear=-1            front=0 rear=0             front=0 rear=1
        (虚拟位置)                  (指向A)                    (指向B)

出队 A (front→1)：         出队 B (front→2)：
┌───┬───┬───┬───┬───┐    ┌───┬───┬───┬───┬───┐
│   │   │   │   │   │    │   │   │   │   │   │     front=2, rear=1 → 空队！
└───┴───┴───┴───┴───┘    └───┴───┴───┴───┴───┘
      ↑     ↑                      ↑     ↑
    front=1 rear=1               front=2 rear=1
                                (front > rear → 判空)
```

| 操作 | 伪代码 | 说明 |
|:---|:---|:---|
| 初始化 | `Q.front = 0; Q.rear = -1` | rear 虚拟指向数组外 |
| 判空 | `Q.front > Q.rear` | 非循环时的判空条件 |
| 判满 | `Q.rear == MAXSIZE - 1` | 非循环时的判满条件 |
| 入队 | `Q.rear = (Q.rear + 1) % MAXSIZE; Q.data[Q.rear] = x` | 先移动 rear，再存入 |
| 出队 | `x = Q.data[Q.front]; Q.front = (Q.front + 1) % MAXSIZE` | 同方式一 |
| 读队头 | `x = Q.data[Q.front]` | 同方式一 |
| 元素个数 | `Q.rear - Q.front + 1` | — |

> ⚠️ 方式二在普通顺序队列中能用，但在**循环队列**场景下，`rear` 可能绕回导致 `rear < front`，此时判空/判满/元素个数都需要重新定义，考研中很少采用。

###### 方式三（少见）：`front = rear = MAXSIZE - 1`

> 初始时 `front = rear = MAXSIZE - 1`，入队先移动 `rear` 再存、出队先取再移动 `front`。不推荐，考研极少使用。

##### 6. 四种初始化方式统一对比

| 特性 | 方式一 `f=r=0` | 方式二 `f=0,r=-1` |
|:---|:---:|:---:|
| **rear 含义** | 下一个空位 | 当前队尾元素 |
| **初始值** | `front = 0, rear = 0` | `front = 0, rear = -1` |
| **判空** | `Q.front == Q.rear` | `Q.front > Q.rear` |
| **判满（牺牲单元）** | `(Q.rear + 1) % MAXSIZE == Q.front` | 不适用于循环队列 |
| **入队** | 先存再移 rear | 先移 rear 再存 |
| **出队** | 先取再移 front | 先取再移 front |
| **读队头** | `Q.data[Q.front]` | `Q.data[Q.front]` |
| **元素个数** | `(Q.rear - Q.front + MAXSIZE) % MAXSIZE` | `Q.rear - Q.front + 1` |

> ⚠️ **考研提醒**：408 默认采用**方式一**（`front = rear = 0`，rear 指向队尾下一位置，牺牲一个单元判满）。审题时先看清题目给的初始化值，再推导对应的判空、入队、出队代码！

### （三）链式队列

链式队列（Linked Queue）是用**链表**实现的队列。`front` 指向队头、`rear` 指向队尾。链式队列**不会满**、不需要循环——天然没有假溢出问题。

> **核心设计原则**：入队在 `rear` 端（尾插法）、出队在 `front` 端（头删法），均为 $O(1)$。队头必须设在链表头部（否则出队找前驱退化为 $O(n)$）。

#### 一、带头结点链式队列

带头结点是 408 最常考的形式。头结点不存数据，`front` 永远指向头结点，第一个数据结点由 `front->next` 访问。

##### 1. 核心结构定义

```c
// 队列的结点
typedef struct QNode {
    int data;
    struct QNode *next;
} QNode;

// 链式队列（包含头尾指针）
typedef struct {
    QNode *front;    // 指向队头（带头结点时指向头结点）
    QNode *rear;     // 指向队尾结点
} LinkQueue;
```

##### 2. 分步图解

**① 初始空队列**

```
        ┌────┬──────┐
        │  ? │ NULL │     ← 头结点（data 不使用）
        └───────────┘
         ↑  ↑
    Q.front │ 
          Q.rear

判空：Q.front == Q.rear  → true ✓
```

**② 入队 A（空队列插入第一个元素）**

```
① 新建结点 s(data=A, next=NULL)

② Q.rear->next = s       头结点的 next 指向 s

③ Q.rear = s             rear 移到新结点

        ┌────┬──────┐    ┌────┬──────┐
        │  ? │  ────────→│ A  │ NULL │
        └────┴──────┘    └────┬──────┘
          ↑                ↑
      Q.front            Q.rear
       (头结点)          (队尾=A)

此时队头 = front->next = A，队尾 = rear = A
```

**③ 入队 B（非空队列插入）**

```
① s(data=B, next=NULL)

② Q.rear->next = s       旧尾结点 A 的 next 指向 B

③ Q.rear = s             rear 移到 B

        ┌────┬──────┐    ┌────┬──────┐    ┌────┬──────┐
        │  ? │  ────────→│ A  │  ────────→│ B  │ NULL │
        └────┴──────┘    └────┴──────┘    └────┬──────┘
          ↑               (队头=A)           ↑
      Q.front                             Q.rear
       (头结点)                           (队尾=B)

队头 = front->next = A，队尾 = rear = B
```

**④ 出队 A（正常出队）**

```
① p = Q.front->next      锁定队头结点 A

② Q.front->next = p->next  头结点跳过 A，指向 B

③ x = p->data; free(p)   取出 A，释放内存

        ┌────┬──────┐    ┌────┬──────┐
        │  ? │  ────────→│ B  │ NULL │
        └────┴──────┘    └────┬──────┘
          ↑                ↑
      Q.front            Q.rear
       (头结点)          (队尾=B)

队头变为 B，队尾仍是 B
```

**⑤ 出队最后一个元素（关键边界！）**

```
继续出队 B：

① p = Q.front->next      锁定结点 B

② Q.front->next = p->next  此时 p->next = NULL → 头结点 next 变为 NULL

③ Q.rear == p ?  YES!  ★ 出队的是最后一个元素
   → Q.rear = Q.front     rear 指回头结点，恢复空队

④ free(p)

        ┌────┬──────┐
        │  ? │ NULL │     ← 恢复空队状态
        └────┬──────┘
        ↑    ↑
    Q.front  Q.rear

判空：Q.front == Q.rear → true ✓
```

##### 3. 操作

###### ① 初始化

```c++
void InitQueue(LinkQueue &Q) {
    Q.front = Q.rear = (QNode *)malloc(sizeof(QNode));
    Q.front->next = NULL;       // 头结点的 next 置空
}
```

| 项目 | 内容 |
|:---|:---|
| 初始状态 | `front` 和 `rear` 都指向新创建的头结点 |
| 判空 | `Q.front == Q.rear`（头尾指针重合） |
| 判满 | 不存在（链队不会满） |

###### ② 判空 (QueueEmpty)

```c++
bool QueueEmpty(LinkQueue Q) {
    return Q.front == Q.rear;
}
```

###### ③ 入队 (EnQueue)

```c++
bool EnQueue(LinkQueue &Q, int x) {
    QNode *s = (QNode *)malloc(sizeof(QNode));
    s->data = x;
    s->next = NULL;

    Q.rear->next = s;       // ① 将新结点链到队尾之后
    Q.rear = s;             // ② rear 后移到新结点
    return true;            // 链队不会满
}
```

| 项目 | 内容 |
|:---|:---|
| 时间复杂度 | 严格 $O(1)$ |
| 核心步骤 | `Q.rear->next = s; Q.rear = s`（尾插法） |
| 空队插入 | 无需特殊处理：`Q.rear` 指向头结点，`Q.rear->next = s` 等同于 `front->next = s` |

###### ④ 出队 (DeQueue)

```c++
bool DeQueue(LinkQueue &Q, int &x) {
    if (Q.front == Q.rear)          // 判空
        return false;

    QNode *p = Q.front->next;       // ① 锁定队头结点（头结点之后第一个）
    x = p->data;                    // ② 取出数据
    Q.front->next = p->next;        // ③ 头结点跳过 p

    if (Q.rear == p)                // ★ 若出队的恰好是最后一个元素
        Q.rear = Q.front;           //    rear 指回头结点，恢复空队

    free(p);                        // ④ 释放结点
    return true;
}
```

| 项目 | 内容 |
|:---|:---|
| 时间复杂度 | 严格 $O(1)$ |
| 核心步骤 | `p = front->next; front->next = p->next; free(p)` |
| 关键边界 | 出队最后一个元素后 `Q.rear = Q.front`，恢复空队 |

###### ⑤ 读队头 (GetHead)

```c++
bool GetHead(LinkQueue Q, int &x) {
    if (Q.front == Q.rear)
        return false;               // 队空
    x = Q.front->next->data;        // 头结点之后第一个结点
    return true;
}
```

#### 二、不带头结点链式队列

不带头结点时 `front` 直接指向第一个数据结点。空队列时 `front = rear = NULL`。入队第一个元素需要**特殊处理**。

##### 1. 结构定义与图解

```
【空队列】
    Q.front → NULL
    Q.rear  → NULL
判空：Q.front == NULL → true ✓

【入队 A, B 后】
    Q.front
       ↓
       ┌────┬──────┐    ┌────┬──────┐
       │ A  │  ────────→│ B  │ NULL │
       └────┴──────┘    └────┬──────┘
        (队头=A)              ↑
                          Q.rear
                          (队尾=B)
```

##### 2. 分步图解

**① 入队 A（空队列——关键！）**

```
空队列：Q.front = Q.rear = NULL

① 新建结点 s(data=A, next=NULL)

② Q.front = Q.rear = s  ★ 两个指针都要指向这个唯一的结点

       Q.front
          ↓
          ┌────┬──────┐
          │ A  │ NULL │
          └────┬──────┘
               ↑
           Q.rear

队头 = 队尾 = A（只有这一个元素）
```

**② 入队 B（非空队列——正常尾插）**

```
① s(data=B, next=NULL)

② Q.rear->next = s       旧尾结点 A 的 next 指向 B

③ Q.rear = s             rear 移到 B

       Q.front
          ↓
          ┌────┬──────┐    ┌────┬──────┐
          │ A  │  ────────→│ B  │ NULL │
          └────┴──────┘    └────┬──────┘
           (队头=A)              ↑
                             Q.rear
                             (队尾=B)
```

**③ 出队最后一个元素（关键边界！）**

```
队列只剩 A：Q.front = Q.rear = 指向 A

① p = Q.front            锁定 A

② x = p->data

③ Q.front = p->next      此时 p->next = NULL → Q.front = NULL

④ Q.rear == p ?  YES!  ★ 出队的正好是最后一个
   → Q.rear = NULL        rear 也置空

⑤ free(p)

    Q.front → NULL
    Q.rear  → NULL

判空：Q.front == NULL → true ✓
```

##### 3. 操作

###### ① 初始化

```c++
void InitQueue(LinkQueue &Q) {
    Q.front = Q.rear = NULL;    // 两个指针都置空
}
```

| 项目 | 内容 |
|:---|:---|
| 初始值 | `Q.front = Q.rear = NULL` |
| 判空 | `Q.front == NULL`（或 `Q.rear == NULL`） |

###### ② 判空 (QueueEmpty)

```c++
bool QueueEmpty(LinkQueue Q) {
    return Q.front == NULL;
}
```

###### ③ 入队 (EnQueue)

```c++
bool EnQueue(LinkQueue &Q, int x) {
    QNode *s = (QNode *)malloc(sizeof(QNode));
    s->data = x;
    s->next = NULL;

    if (Q.front == NULL) {      // ★ 空队列：首尾都指向新结点
        Q.front = Q.rear = s;
    } else {                    // 非空队列：正常尾插
        Q.rear->next = s;
        Q.rear = s;
    }
    return true;                // 链队不会满
}
```

| 项目 | 内容 |
|:---|:---|
| 空队插入 | `Q.front = Q.rear = s`（两指针都指向新结点） |
| 非空插入 | `Q.rear->next = s; Q.rear = s`（同带头结点） |

###### ④ 出队 (DeQueue)

```c++
bool DeQueue(LinkQueue &Q, int &x) {
    if (Q.front == NULL)
        return false;           // 队空

    QNode *p = Q.front;         // ① 锁定队头
    x = p->data;
    Q.front = p->next;          // ② front 后移

    if (Q.rear == p)            // ★ 出队最后一个元素
        Q.rear = NULL;          //    rear 也要置空

    free(p);
    return true;
}
```

| 项目 | 内容 |
|:---|:---|
| 正常出队 | `p = front; front = front->next; free(p)` |
| 出队最后元素 | 额外判断 `Q.rear == p` → `Q.rear = NULL` |

#### 三、带头结点 vs 不带头结点 对比

| 维度 | 带头结点 | 不带头结点 |
|:---|:---|:---|
| 初始值 | `front = rear = malloc(…); front->next = NULL` | `front = rear = NULL` |
| 判空 | `Q.front == Q.rear` | `Q.front == NULL` |
| 入队（空队） | 无需特殊处理 | `Q.front = Q.rear = s`（特殊处理） |
| 入队（非空） | `Q.rear->next = s; Q.rear = s` | 同左 |
| 出队（非最后） | `p = front->next; front->next = p->next; free(p)` | `p = front; front = front->next; free(p)` |
| 出队（最后） | `Q.rear = Q.front; free(p)` | `Q.rear = NULL; free(p)` |
| 内存 | 多一个头结点 | 省一个头结点 |
| **考研倾向** | **★ 408 最常考，逻辑统一** | 需要多种边界判断 |

#### 四、链队 vs 顺序队 对比

| 维度 | 顺序队（循环队列） | 链队 |
|:---|:---|:---|
| 存储方式 | 连续数组 + 取模运算 | 离散结点（单链表） |
| 队头/队尾表示 | `int front, rear`（下标） | `QNode *front, *rear`（指针） |
| 核心问题 | 假溢出 → 循环队列解决 | 无（天然动态） |
| 容量 | 有限（`MAXSIZE - 1`） | 无限（仅受物理内存） |
| 判满 | `(rear+1) % MAXSIZE == front` | 不存在 |
| 判空 | `front == rear` | `front == rear`（带头） / `front == NULL`（不带头） |
| 入队 | $O(1)$（偶尔扩容 $O(n)$） | 严格 $O(1)$（尾插法） |
| 出队 | $O(1)$ | 严格 $O(1)$（头删法） |
| 内存开销 | 无指针开销，可能浪费预分配空间 | 每个结点多一个 `next` 指针 |
| 适用场景 | 元素个数可预估 | 元素个数未知、波动极大 |

### （四）双端队列 (Deque)

#### 一、定义

允许在**两端**都进行插入和删除操作。是栈和队列的泛化形式。

```
示意图：
  出队 ← ┌───┬───┬───┬───┬───┐ → 入队
  入队 → │   │   │   │   │   │ ← 出队
        └───┴───┴───┴───┴───┘
        左端                    右端
```

#### 二、分类

| 类型 | 左端入 | 左端出 | 右端入 | 右端出 | 说明 |
|:---|:---:|:---:|:---:|:---:|:---|
| **双端队列** | ✓ | ✓ | ✓ | ✓ | 全功能，两端均可入可出 |
| **输入受限双端队列** | ✗ | ✓ | ✓ | ✓ | 只能从一端插入，两端均可删除 |
| **输出受限双端队列** | ✓ | ✓ | ✓ | ✗ | 两端均可插入，只能从一端删除 |

#### 三、考研核心考点：合法序列判断

> 给定输入序列 `1, 2, 3, 4`，判断某个输出序列能否由特定类型的双端队列生成。

- **解题关键**：模拟过程，逐个检查目标序列中的元素是否可以从左端或右端合法进入/离开。
- **常见结论**：
  - 普通队列（一端入一端出）：输出序列唯一（即输入序列本身）。
  - 双端队列：合法序列数量远超普通队列，但少于栈（卡特兰数）。
  - 具体题目中需画图模拟，不可记结论。

### （五）栈 vs 队列 对比总结

| 维度 | 栈 (Stack) | 队列 (Queue) |
|:---|:---|:---|
| 原则 | LIFO 后进先出 | FIFO 先进先出 |
| 插入端 | 栈顶 (`top`) | 队尾 (`rear`) |
| 删除端 | 栈顶 (`top`) | 队头 (`front`) |
| 顺序实现核心问题 | 上溢出 (overflow) | **假溢出** (false overflow) |
| 顺序实现解决方案 | 动态扩容 / 共享栈 | **循环队列**（取模运算） |
| 链式实现判空 | 带头结点: `S.top->next == NULL` | 带头结点: `Q.front == Q.rear` |
| 双端版本 | — | 双端队列 (Deque) |
| 经典应用 | 括号匹配、表达式求值、DFS、函数调用 | 层次遍历、BFS、进程调度 |
