### （一）数组的基本概念

##### 1. 定义

**数组（Array）**：由 $n$（$n > 1$）个具有相同数据类型的数据元素 $a_1, a_2, \dots, a_n$ 构成的**有限序列**，是线性表的推广。

| 术语 | 含义 |
|:---|:---|
| **一维数组** | 线性表——每个元素是一个值 |
| **二维数组** | 每个元素受两个下标约束，可看作"线性表的线性表" |
| **$n$ 维数组** | 每个元素受 $n$ 个下标约束 |

##### 2. 数组的性质

| 性质 | 说明 |
|:---|:---|
| 数组元素个数固定 | 一旦定义，**维数和维界不再改变**——只做存取和修改，不做插入/删除 |
| 数组的存储结构 | 采用**顺序存储**（一片连续地址） |
| 随机存取 | 给定下标，可在 $O(1)$ 时间内找到对应元素 |

##### 3. 数组的抽象数据类型

| 操作 | 说明 |
|:---|:---|
| `InitArray(&A, n, bounds)` | 构造一个 $n$ 维数组 |
| `DestroyArray(&A)` | 销毁数组 |
| `Value(A, &e, index)` | 取指定下标元素的值 |
| `Assign(&A, e, index)` | 给指定下标元素赋值 |

### （二）数组的顺序存储

##### 1. 存储方式

多维数组在内存中是一维排列的，有两种映射方式：

| 存储方式 | 排列顺序 | 说明 |
|:---|:---|:---|
| **行优先（Row-major）** | 按行序存储 | C/C++/Pascal 使用 |
| **列优先（Column-major）** | 按列序存储 | Fortran/MATLAB 使用 |

**行优先存储示例**（二维数组 $A[2][3]$）：

```
逻辑结构:              行优先物理存储:
  ┌─────────────┐    [0][0] [0][1] [0][2]  [1][0] [1][1] [1][2]
  │ A00 A01 A02 │    |      ←第0行→       |        ←第1行→      |
  │ A10 A11 A12 │
  └─────────────┘
```

**列优先存储示例**（同上数组）：

```
列优先物理存储:
  [0][0] [1][0] [0][1] [1][1] [0][2] [1][2]
  |  ←第0列→    |    ←第1列→  |   ←第2列→
```

##### 2. 地址计算公式

**一维数组**：$LOC(a_i) = LOC(a_0) + i \times L$（$L$ 为每个元素大小）

**二维数组 $A[m][n]$（行优先）**：

$$LOC(A[i][j]) = LOC(A[0][0]) + (i \times n + j) \times L$$

**二维数组 $A[m][n]$（列优先）**：

$$LOC(A[i][j]) = LOC(A[0][0]) + (j \times m + i) \times L$$

**$n$ 维数组 $A[b_1][b_2]\dots[b_n]$（行优先）**：

$$LOC(A[j_1][j_2]\dots[j_n]) = LOC(A[0]\dots[0]) + \left(\sum_{i=1}^{n} j_i \prod_{k=i+1}^{n} b_k\right) \times L$$

> **408 高频考点**：给定维数和下标，计算二维/三维数组的元素地址（行优先/列优先均可能考）。

##### 3. 实例

```
设 A[3][4] 起始地址为 1000，每个元素占 2 字节：

行优先 LOC(A[2][1]) = 1000 + (2×4 + 1)×2 = 1000 + 18 = 1018
列优先 LOC(A[2][1]) = 1000 + (1×3 + 2)×2 = 1000 + 10 = 1010
```

### （三）特殊矩阵的压缩存储

##### 1. 对称矩阵

$A_{n \\times n}$ 满足 $A[i][j] = A[j][i]$，只需存储**下三角**（含对角线）共 $n(n+1)/2$ 个元素。

**存储示意图**（$n=4$ 对称矩阵，行优先存储下三角）：

```
对称矩阵 A[4][4]:                   压缩存储到一维数组 B[]:
                                    
    ┌───────────────────┐             B[0]  B[1]  B[2]  B[3]  B[4]  B[5]  B[6]  B[7]  B[8]  B[9]
 0  │ a00  a01  a02  a03│           ┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┐
    │   ╲    │    │    ││           │ a00 │ a10 │ a11 │ a20 │ a21 │ a22 │ a30 │ a31 │ a32 │ a33 │
 1  │ a10  a11  a12  a13│           └─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┘
    │   │    ╲    │    ││             ↑     ↑     ↑     ↑     ↑     ↑     ↑     ↑     ↑     ↑
 2  │ a20  a21  a22  a23│           第0行  第1行        第2行             第3行
    │   │    │    ╲    ││           (1个)  (2个)       (3个)             (4个)
 3  │ a30  a31  a32  a33│
    └───────────────────┘             n=4, 存入元素数 = 4×5/2 = 10
     ← 只存左下三角（含对角线）→      对于 i<j: B[k] = A[j][i]（利用对称性）
```

> 下三角部分 $i \\geq j$ 按行存入；上三角部分 $i < j$ 通过 $A[i][j] = A[j][i]$ 映射到下三角的对应位置。

**存储方式**（行优先存储下三角到一维数组 $B$）：

$$
k = \begin{cases}
\dfrac{i(i-1)}{2} + j - 1 & \text{当 } i \geq j \\[8pt]
\dfrac{j(j-1)}{2} + i - 1 & \text{当 } i < j
\end{cases}
$$

##### 2. 三角矩阵

| 类型 | 存储范围 | 元素数 |
|:---|:---|:---|
| **下三角** | 对角线及以下（$i \\geq j$） | $n(n+1)/2$ |
| **上三角** | 对角线及以上（$i \\leq j$） | $n(n+1)/2$ |

**存储示意图**（$n=4$ 下三角矩阵，末尾存常量 $c$）：

```
下三角矩阵 A[4][4]:                 压缩存储到一维数组 B[]:
                                    
    ┌───────────────────┐           B[0]  B[1]  B[2]  B[3]  B[4]  B[5]  B[6]  B[7]  B[8]  B[9]   B[10]
 0  │ a00   c    c    c │         ┌─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┬─────┬──────┐
    │   ╲    │    │     │         │ a00 │ a10 │ a11 │ a20 │ a21 │ a22 │ a30 │ a31 │ a32 │ a33 │  c   │
 1  │ a10  a11   c    c │         └─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┴─────┴──────┘
    │   │    ╲    │     │           ↑     ↑     ↑     ↑     ↑     ↑     ↑     ↑     ↑     ↑      ↑
 2  │ a20  a21  a22   c │        第0行   第1行        第2行             第3行                  上三角常量
    │   │    │    ╲     │        (1个)  (2个)        (3个)             (4个)                  (存1个c)
 3  │ a30  a31  a32  a33│
    └───────────────────┘          n=4, 存入元素数 = n(n+1)/2 + 1 = 11
     ← 下三角 + 对角线 →             B[10] 存放上三角常数 c
        上三角全为常数 c
```

> 下三角矩阵的 $k$ 映射同对称矩阵公式（$i \\geq j$），上三角部分统一存一个常数 $c$ 到 $B[n(n+1)/2]$。

**上三角矩阵**同理，行优先存储上三角（含对角线）+ 最后一个位置存下三角常数。

##### 3. 对角矩阵（三对角矩阵）

所有非零元素集中在以主对角线为中心的**三条对角线**上，即 $|i-j| > 1$ 时 $A[i][j] = 0$。

**存储示意图**（$n=5$ 三对角矩阵）：

```
三对角矩阵 A[5][5]:                    压缩存储到一维数组 B[]:
                                       
    ┌────────────────────────┐          B下标映射:
 0  │ a00  a01   0    0    0 │          第0行: a00 a01             ↦ B[0] B[1]
    │   ╲    ╲   │    │    │ │          第1行: a10 a11 a12          ↦ B[2] B[3] B[4]
 1  │ a10  a11  a12   0    0 │          第2行: a21 a22 a23          ↦ B[5] B[6] B[7]
    │   ╲    ╲    ╲   │    │ │          第3行: a32 a33 a34          ↦ B[8] B[9] B[10]
 2  │  0   a21  a22  a23   0 │          第4行: a43 a44             ↦ B[11] B[12]
    │   │    ╲    ╲    ╲   │ │
 3  │  0    0   a32  a33  a34│           n=5, 非零元 = 3n-2 = 13
    │   │    │    │╲    ╲    ╲│          每行最多3个元素, 按行优先依次存入B[]
 4  │  0    0    0   a43  a44│
    └────────────────────────┘
    三对角线: ╲ (主对角线)
              ╲ (次对角线)
            只有三对角线上元素非零
```

> 映射公式：$k = 2i + j - 3$（当 $|i-j| \\leq 1$）。三对角矩阵首行（$i=0$）和尾行（$i=n-1$）各只有 2 个非零元，中间行各 3 个。

### （四）稀疏矩阵

##### 1. 定义

非零元素个数远小于矩阵元素总数（通常 $< 5\\%$）的矩阵。

##### 2. 三元组表示法

将每个非零元素表示为一个三元组 $(i, j, value)$，所有三元组按行序排列组成一个线性表。

```c
#define MAXSIZE 1000

typedef struct {
    int i, j;        // 行号、列号
    ElemType e;      // 元素值
} Triple;

typedef struct {
    Triple data[MAXSIZE + 1];
    int mu, nu, tu;  // 行数、列数、非零元个数
} TSMatrix;
```

**转置算法**（两种）：

| 方法 | 时间 | 说明 |
|:---|:---|:---|
| 直接转置 | $O(nu \\times tu)$ | 扫描 $nu$ 次，每趟将同列元素填入 |
| 快速转置 | $O(nu + tu)$ | 先统计每列元素数，确定位置后一次填入 |

##### 3. 十字链表表示法

每个非零元素作为一个结点，同时挂在**行链表**和**列链表**中。

```c
typedef struct OLNode {
    int i, j;
    ElemType e;
    struct OLNode *right, *down;  // 行指针、列指针
} OLNode;

typedef struct {
    OLNode *rhead[MAXSIZE], *chead[MAXSIZE];
    int mu, nu, tu;
} CrossList;
```

> 十字链表适合**频繁进行矩阵运算**（如加法、乘法），插入/删除非零元灵活。

### （五）广义表

##### 1. 定义

**广义表（Generalized List）**：线性表的推广——表中元素可以是**原子**（单个数据元素），也可以是**另一个广义表**（子表）。

记作：$LS = (a_1, a_2, \\dots, a_n)$

| 术语 | 含义 | 示例 |
|:---|:---|:---|
| 原子 | 不可再分的数据元素 | `A = (a, b, c)` |
| 子表 | 作为元素的广义表 | `B = (a, (b, c))` |
| 表头 Head | 第一个元素 $a_1$ | `Head(A) = a` |
| 表尾 Tail | 除第一个元素外的其余元素组成的表 | `Tail(A) = (b, c)` |
| 深度 | 括号嵌套的最大层数 | `(a, (b, (c)))` 深度为 3 |
| 长度 | 最外层元素个数 | `(a, (b, c))` 长度为 2 |

##### 2. 广义表的特点

| 特点 | 说明 |
|:---|:---|
| **递归定义** | 子表本身也是广义表 |
| **可以共享** | 多个广义表可共享同一个子表 |
| **可以递归** | 广义表可以是自身的子表（需通过指针） |
| **元素有序** | 表中元素有线性次序 |

##### 3. 广义表的存储

```c
typedef enum { ATOM, LIST } TagType;

typedef struct GLNode {
    TagType tag;              // 标志：原子 or 子表
    union {
        ElemType data;        // tag=ATOM：原子值
        struct {
            struct GLNode *hp, *tp;  // tag=LIST：表头指针、表尾指针
        } ptr;
    };
} GLNode;
```

> 头尾表示法：每个结点包含 `tag`、`hp`（指向表头）、`tp`（指向表尾），形成链式结构。

##### 4. 广义表举例

```
A = ()             空表，长度 0，深度 1
B = (e)            单元素表，长度 1，深度 1
C = (a, (b, c))    长度 2，深度 2
D = ((a, b), c)    长度 2，深度 2
E = (a, E)         递归表，无限深度
```
