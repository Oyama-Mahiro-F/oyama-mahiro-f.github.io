### 一、 数据结构的基本概念

* **数据**：是信息的载体，是描述客观事物属性的数、字符及所有能输入到计算机中并被计算机程序识别和处理的符号的集合 。


* **数据元素与数据项**：数据元素是数据的基本单位，通常作为一个整体进行考虑和处理 。一个数据元素可由若干数据项组成，数据项是构成数据元素的不可分割的最小单位，对客观事物某一方面的数据描述 。


* **数据对象**：数据对象则是具有相同性质的数据元素的集合，是数据的一个子集 。


* **数据类型**：是一个值的集合和定义在此集合上的一组操作的总称 。
    * 原子类型：其值不可再分的类型
    * 结构类型：由若干个数据项组成的类型
* **抽象数据类型 (ADT)**：是抽象数据组织及与之相关的操作，它用数学化的语言定义数据的逻辑结构和运算，与具体的实现无关。

    * ADT=(D,S,P)

        * D：数据对象
        * S：D上的关系集
        * P：对D的基本操作集

* **数据结构**：是指相互之间存在一种或多种特定关系的数据元素的集合，它是数据类型及其相互关系的集合 。包含**逻辑机构、存储结构和数据的运算**。


### 二、 数据结构的三要素

数据结构主要由以下三个要素构成：

1. #### **逻辑结构**（数据元素之间的逻辑关系）：

* **集合**：各个元素同属一个集合，别无其他关系 。
* **线性结构**：数据元素之间是一对一的关系 。
* **树形结构**：数据元素之间是一对多的关系 。
* **图状结构（网状结构）**：数据元素之间是多对多的关系 。

2. #### **物理结构 / 存储结构**（数据元素的表示及其相互的关系）：

* **顺序存储**：逻辑上相邻的元素存储在物理位置上也相邻的存储单元中 。
* **链式存储**：逻辑上相邻的元素在物理位置上可以不相邻，借助指示元素存储地址的指针来表示元素之间的逻辑关系 。
* **索引存储**：在存储元素信息的同时，还建立附加的索引表 。
* **散列存储**：根据元素的关键字直接计算出该元素的存储地址，又称哈希 (Hash) 存储 。

3. #### **数据的运算**：

* 包括运算的**定义**和**实现**。运算的定义针对逻辑结构，指出运算的功能；运算的实现针对存储结构，指出运算的具体操作步骤 。

### 三、 算法的基本概念

* **核心理念**：程序 = 数据结构 + 算法 。算法是对特定问题求解步骤的一种描述，它是指令的有限序列，其中的每条指令表示一个或多个操作 。

- **算法的五个特性**：

  1. **有穷性**：一个算法必须总在执行有穷步之后结束，且每一步都可在有穷时间内完成（注：程序可以是无穷的） 。

  1. **确定性**：算法中每条指令必须有确切的含义，相同的输入只会产生相同的输出 。

  3. **可行性**：算法中描述的操作都可以通过已经实现的基本运算执行有限次来实现 。
  
  
     4. **输入**：一个算法有零个或多个输入 。
  
  
     5. **输出**：一个算法有一个或多个输出 。
  

* **“好”算法的特质**：
  
    1. **正确性**：能够正确地解决求解问题 。
    
    2. **可读性**：具有良好的可读性，以帮助人们理解 。
    
    3. **健壮性**：输入非法数据时，算法能适当地做出反应或进行处理 。
    
     4. **通用性**：算法的处理结果对于一般的数据都成立
    
     5. **高效率与低存储量需求**：即算法执行省时、省内存（时间复杂度低、空间复杂度低） 。

### 四、 算法的复杂度度量

1. **时间复杂度**：衡量算法时间开销 $T(n)$与问题规模 $n$的关系 。

* **计算规则**：
    * **加法规则**（保留最高阶）：$T(n) = O(f(n)) + O(g(n)) = O(max(f(n), g(n)))$。
    
    - **乘法规则**：$T(n) = O(f(n)) \times O(g(n)) = O(f(n) \times g(n))$。


* **杂度排序（常对幂指阶）**：$O(1) < O(log_2 n) < O(n) < O(n log_2 n) < O(n^2) < O(n^3) < O(2^n) < O(n!) < O(n^n)$。


* **三种时间复杂度**：
    * **最好时间复杂度**：考虑输入数据“最好”的情况 。
    
    
    * **最坏时间复杂度**：考虑输入数据“最坏”的情况 。
    
    
    * **平均时间复杂度**：考虑所有输入数据都等概率出现的情况下的期望运行时间 。
    

#### 2. 空间复杂度

衡量算法空间开销（内存开销）与问题规模 $n$之间的关系 $S(n)$。

* **算法原地工作**：无论问题规模怎么变，算法运行所需的内存空间都是固定的常量，即 $S(n) = O(1)$。


* **内存分析**：只需关注与问题规模相关的变量（如一维数组 $O(n)$，二维数组 $O(n^2)$） 。


* **函数递归调用**：递归程序的空间复杂度通常等于递归调用的深度 。
