<h2 align="center">定语从句</h2>

**定语从句**（Attributive Clause）是英语语法中最重要、也是最常用的复合句结构之一。它的作用相当于一个形容词，用来修饰主句中的名词或代词。

> **基本结构：** 先行词 + 关系词 + 定语从句
>
> **例句：** The man **who is speaking** is my teacher. （正在说话的那个男人是我的老师。）

### （一）核心概念

#### 一、定义

要完全理解定语从句，首先需要掌握两个核心概念：

- **先行词（Antecedent）：** 被定语从句修饰的名词或代词。（例如上方例句中的 *The man*）
- **关系词（Relative Word）：** 引导定语从句的词。它起到连接主句和从句的作用，代替先行词，并在从句中充当特定的句子成分。（例如上方例句中的 *who*）
  - 选择哪种关系词，取决于**先行词指代的是什么**（人、物、时间、地点等），以及**关系词在从句中充当什么成分**。关系词分为以下两大类：**关系代词和关系副词**

选择关系代词还是关系副词，是定语从句中最核心、也最容易出错的考点。

很多同学做题时习惯“看前面”：看到表示时间的词（the day）就填 `when`，看到表示地点的词（the place）就填 `where`。**这是绝对错误的！**

真正的黄金法则是“**看后面**”——**看定语从句内部缺什么成分。**
### （二）关系代词和关系副词的选择
#### 核心判定法则：从句“缺不缺”名词

在判断之前，先把主句和先行词放在一边，只看定语从句这部分的结构：

1. **缺主语或宾语 $\rightarrow$ 必须用关系代词 (which, that, who 等)。**

   因为代词本质上就是名词的替身，它要在从句里当“骨架”。

2. **主谓宾完整，什么都不缺 $\rightarrow$ 必须用关系副词 (when, where, why)。**

   因为副词只是补充说明时间、地点、原因的“修饰语”，加不加都不影响句子的基本骨架。

#### 经典对比：同一个先行词，不同的关系词

为了让你直观感受，我们拿同一个先行词来造句对比。这也是各类英语考试中最爱设下的“陷阱”。

##### 1. 先行词是“地点” (The lab)

- **句子 A：** This is the lab ______ we built last year.
  - **分析：** 看从句 `we built last year`。谓语动词 build 是及物动词，后面必须跟东西（建了什么？）。这里**缺宾语**。
  - **答案：** **which / that** （代词充当 built 的宾语）
- **句子 B：** This is the lab ______ we conduct penetration tests.
  - **分析：** 看从句 `we conduct penetration tests`（我们进行渗透测试）。主语 (we)、谓语 (conduct)、宾语 (tests) 齐全，**结构完整，什么都不缺**。它只缺一个表示地点的状语。
  - **答案：** **where** （或 in which）

##### 2. 先行词是“时间” (The days)

- **句子 A：** I will never forget the days ______ we spent together.
  - **分析：** 看从句 `we spent together`。谓语动词 spend 后面缺宾语（度过了什么？）。**缺宾语**。
  - **答案：** **which / that**
- **句子 B：** I will never forget the days ______ we worked on the ctfSolver project.
  - **分析：** 看从句 `we worked on the ctfSolver project`。主干完整（我们开发项目）。**不缺主宾**，只缺时间状语。
  - **答案：** **when** （或 in/during which）

##### 3. 先行词是“原因” (The reason)

- **句子 A：** The reason ______ he explained to me is convincing.
  - **分析：** 从句 `he explained to me` 中，explain 是及物动词（解释了什么？）。**缺宾语**。
  - **答案：** **which / that**
- **句子 B：** The reason ______ he was late is obvious.
  - **分析：** 从句 `he was late` 是主系表结构，本身已经**完整**。
  - **答案：** **why** （或 for which / that）

#### 终极解题三步法（心法口诀）

以后遇到填空题或自己写作文时，按照这个逻辑走，绝对不会错：

1. **找先行词：** 确定被修饰的名词是什么。
2. **看从句缺不缺：** 把先行词代入到从句里面去。
   - 如果它能在从句里作主语或宾语 $\rightarrow$ **选代词 (which/that/who)**。
   - 如果要在它前面加个介词（如 in, on, at），从句才通顺 $\rightarrow$ **选副词 (where/when/why)**。
3. **验算（介词还原法）：** 所有的关系副词都可以还原成 `介词 + which`。
   - where = in/at/on which
   - when = in/on/during which
   - why = for which

> **程序员思维类比：**
>
> 如果定语从句是一个函数 `Function(x)`：
>
> - 如果你发现函数签名里少了一个必需的参数（比如缺主语/宾语），那关系词就是这个必传的参数变量（**代词**）。
> - 如果你发现函数内部逻辑闭环，参数齐全，那你传进去的仅仅是一个可选的环境配置项（环境变量：时间、地点），那就是**副词**。


### （二）关系代词
#### 一、常见关系代词

关系代词在定语从句中通常作**主语、宾语、表语或定语**。

| **关系代词** | **先行词指代** | **在从句中充当的成分**   | **例句**                                            |
| ------------ | -------------- | ------------------------ | --------------------------------------------------- |
| **who**      | 人             | 主语、宾语               | The boy **who** broke the window is Tom.            |
| **whom**     | 人             | 宾语（作宾语时常可省略） | The person **(whom)** you are waiting for has left. |
| **which**    | 物             | 主语、宾语               | The book **which** I bought is very interesting.    |
| **that**     | 人 / 物        | 主语、宾语、表语         | A plane is a machine **that** can fly.              |
| **whose**    | 人 / 物        | 定语（表示“……的”）       | I know the girl **whose** mother is a doctor.       |

#### 二、that和which

在定语从句中，当先行词是**物**时，`that` 和 `which` 经常可以互换使用。但在某些特定语法场景下，它们有着严格的界限。

记住一个大原则：**绝大多数情况下找 `that`，但遇到“逗号”和“介词”找 `which`。**


##### 1. 只能用 `which` 的情况

当遇到以下两种情况时，关系代词**绝对不能**用 `that`，只能用 `which`：

**（1）引导非限制性定语从句（有逗号隔开）**

如果定语从句前有逗号，说明这是一个补充说明的从句，必须用 `which`。

- **错：** The book, *that* I bought yesterday, is great.
- **对：** The book, **which** I bought yesterday, is great.

**（2）放在介词的后面（介词提前）**

当介词被提前到关系代词前面时，只能用 `which`。

- **错：** This is the room in *that* he lives.

- **对：** This is the room **in which** he lives.

  *(注：如果介词不提前，放在句末，则用 `that` 或 `which` 都可以：This is the room that/which he lives in.)*

##### 2. 只能用 `that` 的情况

这是各类英语考试中最常考的考点。当先行词出现以下 5 种特征时，**只能用 `that`，不能用 `which`**：

**（1）先行词同时包含“人”和“物”**

如果前面的词既有人又有物，为了公平起见，只能用中立的 `that`。

- **例句：** They talked about the **people and things that** they saw on the way.

**（2）先行词是不定代词**

当先行词是 *all, much, anything, everything, nothing, little, none, the one* 等不定代词时。

- **例句：** Is there **anything that** I can do for you?

**（3）先行词被最高级或序数词修饰**

当先行词前面有形容词最高级（the best, the most...）或序数词（the first, the second...）修饰时。

- **例句：** This is the **best** movie **that** I have ever seen.
- **例句：** He is the **first** person **that** passed the exam.

**（4）先行词被特定的强调词修饰**

当先行词前有不定限定词*some*、*any*、*no*、*much*、*all*、*the very*（正是）、*the only*（唯一的）、*the same*（同样的）、*the last*（最后的）等词修饰时。

- **例句：** This is **the only** book **that** I want to read.

**（5）主句已经是以 `which` 开头的特殊疑问句**

为了避免重复，如果主句用了 *which* 提问，定语从句就用 `that`。

- **例句：** **Which** is the car **that** killed the dog?

> **核心总结技巧：**
>
> 做题或写作时，先看先行词是什么。
>
> 1. 看到**逗号**或**介词在前面** $\rightarrow$ 选 `which`。
> 2. 看到**人+物、不定代词、最高级、序数词、the only/very** $\rightarrow$ 选 `that`。
> 3. 普通的物，没有上述特殊标志，且没有逗号 $\rightarrow$ `that` 和 `which` 随便用。

### （三）关系副词

如果定语从句中主谓宾结构完整，关系词在从句中作**状语**（表示时间、地点、原因），则需要使用关系副词。
#### 一、常见关系副词
| **关系副词** | **先行词指代**     | **相当于**   | **例句**                                           |
| ------------ | ------------------ | ------------ | -------------------------------------------------- |
| **when**     | 时间               | 介词 + which | I will never forget the day **when** we met.       |
| **where**    | 地点               | 介词 + which | This is the house **where** I lived ten years ago. |
| **why**      | 原因 (常为 reason) | for + which  | I don't know the reason **why** he was late.       |

#### 二、that代替关系副词

在定语从句中，`that` 真的是一个“超级替身”。它不仅能代替关系代词（who, whom, which），在**特定的情况下**，它还能跨界代替关系副词（when, where, why），甚至连 `that` 本身都可以被完全省略。

但这有一个极其严格的先决条件：**先行词必须是那些最常见、最泛指的“时间、地点、原因、方式”名词。**

以下是 `that` 代替关系副词的四大核心场景：

##### 1. 替代 `why` (表原因)

当先行词是 **the reason** 时，引导定语从句的关系副词本应是 `why`（或 `for which`），但在非正式场合，常常用 `that` 来代替，或者干脆省略。

- **常规用法：** This is the reason **why / for which** the system crashed.
- **that 替代：** This is the reason **that** the system crashed.
- **直接省略：** This is the reason **[省略]** the system crashed.

##### 2. 替代 `when` (表时间)

当先行词是泛指时间的词，如 **time, day, year, moment, night** 等时，可以用 `that` 代替 `when`（或 `介词 + which`）。

- **常规用法：** I will never forget the day **when / on which** we first met.

- **that 替代：** I will never forget the day **that** we first met.

- **直接省略：** I will never forget the day **[省略]** we first met.

  *(注：如果先行词是很具体的时间段，比如 "the 1990s"，则通常不建议用 that 替代。)*

##### 3. 替代 `where` (表地点)

当先行词是泛指地点的词，最常见的是 **place**，或者复合不定代词如 **anywhere, everywhere, somewhere** 时，可以用 `that` 代替 `where`。

- **常规用法：** We need to find a secure place **where** we can store the backup data.
- **that 替代：** We need to find a secure place **that** we can store the backup data.
- **直接省略：** We need to find a secure place **[省略]** we can store the backup data.

##### 4. 终极考点：替代表示“方式”的词 (Way)

这是英语考试中最喜欢设坑的地方。当先行词是 **the way**（方式、方法）时，**英语中绝对没有 "the way how" 这种搭配**。

要引导修饰 "the way" 的定语从句，只有以下三种合法途径，其中 `that` 扮演了极其重要的角色：

| **引导方式**                                                 | **语法地位**   | **例句**                                                     |
| ------------------------------------------------------------ | -------------- | ------------------------------------------------------------ |
| **in which**                                                 | 最正式、严谨   | The way **in which** you configured the firewall is perfect. |
| **that**                                                     | 较日常、口语   | The way **that** you configured the firewall is perfect.     |
| **直接省略**                                                 | 最常用、最地道 | The way **[省略]** you configured the firewall is perfect.   |
| *(错误示范：The way how you configured the firewall is perfect. —— 绝对不要这么写！)* |                |                                                              |

> **核心总结：**
>
> 以后在写作或翻译中，只要你用到了 **the reason, the time/day, the place, the way** 这四个高频词作先行词，你不仅可以用 `that` 去引导后面的从句，还可以直接把引导词**省掉不写**，这样写出来的句子既简洁，又显得地道高级。

#### 三、when和where引导定语从句和状语从句的区别

区分 `when` 和 `where` 引导的是定语从句还是状语从句，最核心的标尺只有一个：**看它们前面有没有先行词（被修饰的名词）。**

> **一句话总结：**
>
> - **定语从句 = 形容词**。它前面**必须**有一个表示时间或地点的名词（先行词），从句的作用是解释“哪一个”时间或地点。
> - **状语从句 = 副词**。它前面**没有**需要修饰的名词，它直接修饰主句的谓语动词，说明主句动作发生的“时间”或“地点”。

##### 1. `when` 的对比

| **句子类型** | **核心特征**                     | **典型例句**                                               | **翻译技巧**                                                 |
| ------------ | -------------------------------- | ---------------------------------------------------------- | ------------------------------------------------------------ |
| **定语从句** | 前面有表时间的**名词\代词**      | I will never forget **the day** *when* we met.             | 翻译为“……**的**”。（我永远不会忘记我们相遇**的**那一天。）   |
| **状语从句** | 前面没有表时间的名词，直接接主句 | I was running an exploit script *when* the server crashed. | 翻译为“**当**……**时**”。（**当**服务器崩溃**时**，我正在运行漏洞利用脚本。） |

**深度辨析：**

在定语从句中，`when` 相当于 `on/in/at which`，它指代前面的 "the day"。而在状语从句中，`when` 就是一个纯粹的时间连词，连接两个动作。

##### 2. `where` 的对比

| **句子类型** | **核心特征**                | **典型例句**                                                 | **翻译技巧**                                                 |
| ------------ | --------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **定语从句** | 前面有表地点的**名词\代词** | This is **the lab** *where* we conduct penetration tests.    | 翻译为“……**的**”。（这是我们进行渗透测试**的**实验室。）     |
| **状语从句** | 前面没有表地点的名词        | You should save the configuration file *where* it is easy to find. | 翻译为“在……**的地方**”。（你应该把配置文件保存在容易找到**的地方**。） |

**深度辨析：**

在定语从句中，`where` 相当于 `in which`，指代前面的 "the lab"。而在状语从句中，`where` 前面紧挨着的是名词 file，但 `where` 并不修饰 file（配置文件并不是一个地点），它修饰的是动词 "save"，说明“保存在哪里”。

##### 3. 简单的方法

如果你在阅读长难句时感到困惑，可以用这两步来检验：

1. **“名词”测试：** 圈出 `when/where`，看它左边紧挨着的词。如果是个动词，那100%是状语从句。如果是个表示时间/地点的名词，大概率是定语从句。
2. **“剥离”测试：**
   - **状语从句**把 `when/where` 所在的句子整个砍掉，主句依然是一个意思完整、独立存在的句子。
   - **定语从句**如果砍掉，主句虽然语法可能完整，但往往会让人觉得话没说完（比如砍掉后变成 "I will never forget the day." —— 听者会问：哪一天？）。

### （四）限制性与非限制性定语从句

根据定语从句与主句关系的紧密程度，定语从句分为两类：

- **限制性定语从句：** 与先行词关系极为密切，是句子不可或缺的部分。如果去掉，主句意思就不完整或不明确。**（不用逗号隔开）**
  - *例句：* I have a brother **who is a doctor**. （我有一个当医生的哥哥。——暗示我可能还有其他哥哥，从句起到了限制和区分的作用。）
- **非限制性定语从句：** 仅仅对先行词起补充说明作用。如果去掉，主句意思仍然完整。**（必须用逗号与主句隔开，且不能用 `that` 引导）**
  - *例句：* I have a brother, who is a doctor. （我只有一个哥哥，顺便提一下他是当医生的。）

- **找准先行词：** 翻译和做题的第一步，是明确从句到底在修饰前面的哪个名词或代词。
- **分析从句缺什么成分：**
  - 如果从句**缺主语或宾语**，必须用关系代词（who, which, that）。
  - 如果从句**主谓宾完整**，缺时间、地点或原因，必须用关系副词（when, where, why）。
- **注意介词提前的情况：** 关系代词作介词的宾语时，介词可以放在关系代词之前（例如 *in which*, *to whom*）。**注意：** 当介词提前时，关系代词只能用 `which`（指物）或 `whom`（指人），绝对不能用 `that` 或 `who`。

### （五）特殊的关系代词

在常规的 `who, whom, whose, which, that` 之外，英语中还有几个特殊的关系代词。它们在长难句（尤其是考研英语阅读和翻译）中出现频率极高，往往带有特殊的搭配或隐含的逻辑关系。

最需要掌握的特殊关系代词有三个：**`as`**、**`but`** 和 **`than`**。此外，还有一个经常冒充定语从句的“伪装者”。

#### 1. 最灵活：`as`

`as` 引导定语从句时，既可以引导限制性定语从句，也可以引导非限制性定语从句。

##### 场景 A：固定搭配（限制性定语从句）

当主句中出现了 `such`, `the same`, `as` 时，后面的定语从句**必须**用 `as` 来引导，形成固定呼应。

- **such ... as ...** （像……这样的）
  - *例句：* You should read **such** books **as** are recommended by the professor. （你应该读教授推荐的这类书。）
  - *注意：* 这里的 `as` 在从句中作主语。绝对不能写成 `such books that/which...`。
- **the same ... as ...** （和……同样的）
  - *例句：* We are using **the same** encryption algorithm **as** we used last year. （我们正在使用和去年相同的加密算法。）

##### 场景 B：指代整个句子（非限制性定语从句）

`as` 也可以像 `which` 一样，引导一个逗号隔开的非限制性定语从句，指代**主句的整件事**。

- **与 which 的核心区别：**
  1. **位置灵活：** `which` 只能放在主句后面；而 `as` 引导的从句可以放在句首、句中或句末。
  2. **词义色彩：** `as` 通常带有“正如……”、“就像……”的意思（常与 know, see, expect, point out 等词连用）；而 `which` 只是客观陈述（常译为“这”）。
- *例句：* **As** is known to all, TCP is a reliable transmission protocol. （众所周知，TCP 是一种可靠的传输协议。） —— *这里只能用 as，不能用 which，因为它在句首。*

#### 2. 隐含否定：`but`

这是一个极其高级的用法，如果在写作中用出来，绝对是加分项。`but` 引导定语从句时，实际上相当于 **`that/who...not`**。

**触发条件：** 主句往往是否定句（包含 no, not, little, few 等），`but` 引导从句，形成“双重否定表肯定”的强烈语气。

- *例句：* There is **no** rule **but** has exceptions.
- *还原逻辑：* There is **no** rule **that does not** have exceptions.
- *翻译：* 任何规则都有例外。（没有哪条规则是没有例外的。）

#### 3. 比较级：`than`

当主句中出现比较级（`more`, `-er`）时，`than` 可以作为关系代词引导定语从句，并在从句中充当主语或宾语。

- *例句：* Don't assign the server **more** tasks **than** are necessary.
- *分析：* `than` 在这里指代 tasks，并在从句中作主语（than are necessary）。
- *翻译：* 不要给服务器分配比必需数量更多的任务。

#### ⚠️ 绝对预警：伪装者 `what`

在复习定语从句时，必须牢记一条铁律：**`what` 永远不能引导定语从句！**

定语从句的前提是前面必须有一个“名词（先行词）”让它去修饰。而 `what` 自身内部就已经包含了名词，它等于 `the thing that`。因此，`what` 只能引导**名词性从句**（主语从句、宾语从句、表语从句）。

- **错误示范：** I didn't understand the **code what** he wrote. （code 是先行词，不能用 what）
- **正确改法 1（定语从句）：** I didn't understand the **code that/which** he wrote.
- **正确改法 2（宾语从句）：** I didn't understand **what** he wrote. （what 直接做 understand 的宾语，意思是“他写的东西”）

### （六）介词+which\whom

**“介词 + which / whom”** 是英语高级语法中的重头戏。在考研英语的阅读长难句、翻译，以及专业的学术论文中，这种结构几乎无处不在。

它的本质其实很简单：**就是把原本放在从句句末的介词，提到了关系代词的前面。** 这样做的目的是为了避免句子以介词结尾（在传统正式英语中被认为不够严谨），使句子结构看起来更紧凑、更正式。

#### 一、根本区别

一旦介词被提前，它后面的关系代词立刻进入“VIP 严格准入模式”：

- 如果先行词是**物**，只能用 **`which`**。
- 如果先行词是**人**，只能用 **`whom`**。
- **绝对预警：** 介词后面**永远、绝对、绝对不能**接 `that` 或 `who`！

> **对比：**
>
> - *日常口语：* The man **who/that** I was talking **to** is my tutor.
> - *考研/学术写作：* The man **to whom** I was talking is my tutor. （介词 to 提前，who/that 必须变成 whom）

#### 二、介词到底该选哪一个？

很多同学面对填空或翻译时，知道要用 which/whom，但不知道前面的介词该填 in, on, at, for 还是 to。寻找正确介词有以下 3 个核心方法：

##### 方法 1：看从句里的动词/形容词搭配（最常见）

介词往往是从句里的谓语动词或形容词“遗落”下来的。你只需要看从句里的动词习惯跟什么介词搭配。

- **例句：** This is the target server ______ the script is attacking.
  - **分析：** 从句谓语是 attack，但 attack 是及物动词，不带介词。换个思路，如果我们要表达“依赖”，动词是 depend，固定搭配是 **depend on**。
  - **造句：** This is the target server **on which** our automated testing depends. （这就是我们自动化测试**所依赖的**目标服务器。）

##### 方法 2：看前面的先行词搭配

有时候从句里的动词没有固定的介词搭配，这时候就要看前面的先行词（名词）习惯和什么介词连用。

- **例句：** We need to locate the exact memory address ______ the buffer overflow occurs.
  - **分析：** 发生溢出的地方是“内存地址 (memory address)”。在英语中，表达“在某个特定的地址/点”，介词用 **at**。
  - **答案：** We need to locate the exact memory address **at which** the buffer overflow occurs.

##### 方法 3：根据句子的逻辑意思（高级用法）

有些介词既不取决于动词，也不完全取决于名词，而是由上下句的逻辑关系（如“没有它”、“凭借它”、“除此之外”）决定的。

- **表“没有”：** We must secure the private key, **without which** the data cannot be decrypted. （我们必须保护好私钥，**没有它**数据就无法解密。）
- **表“凭借/通过”：** They developed a new hashing algorithm **by which** file integrity can be verified. （他们开发了一种新的哈希算法，**通过该算法**可以验证文件完整性。） *注：这里的 by which 就是我们之前提到的 whereby。*

#### 终极公式：“介词 + which” 与关系副词的转换

正如前面提到的，关系副词在底层逻辑上其实就是“介词 + which”的封装。在很多情况下，它们可以无缝互换，这也是考研英语中最喜欢考的同义替换：

| **关系副词**       | **对应的“介词 + which”**          | **典型例子**                                         |
| ------------------ | --------------------------------- | ---------------------------------------------------- |
| **where** (表地点) | **in / at / on + which**          | the lab **in which** (where) we test vulnerabilities |
| **when** (表时间)  | **in / on / at / during + which** | the year **in which** (when) he takes the exam       |
| **why** (表原因)   | **for + which**                   | the reason **for which** (why) the code failed       |

> **写作提分建议：**
>
> 在准备 11408 等高难度考试的英语部分时，如果在作文里想写 `when` 或 `where`，刻意把它替换成 `in which` 或 `at which`，会让你的句子立刻透出一种严谨的学术感。

### （七）分隔式定语从句

你指的应该是**分隔式定语从句**。这是英语长难句（尤其是考研英语阅读和翻译）中最具杀伤力的“隐形刺客”。

按照正常逻辑，定语从句应该像强力胶一样，死死贴在它修饰的名词（先行词）后面。但**当先行词后面的谓语或状语非常短，而定语从句又特别长时**，为了避免句子出现“头重脚轻”的尴尬局面，英语母语者会把谓语或状语提到前面，硬生生把先行词和定语从句拆散。

这就形成了：**先行词 + 插入成分（谓语/状语） + 关系词 + 定语从句**。

#### 一、为什么会产生分隔？（英语的尾重原则）

我们先看一个糟糕的句子排版：

> *头重脚轻（不地道）：* **An automated agent** *which can automatically execute buffer overflow attacks and detect format string vulnerabilities* **was designed**.

这个句子主语长达十几个词，谓语却只有短短的 "was designed"，读起来喘不过气。于是，语法规则允许我们将 "was designed" 提前：

> *分隔式（地道）：* **An automated agent** was designed *which can automatically execute buffer overflow attacks and detect format string vulnerabilities*.
>
> *(设计了一个自动化代理，它可以自动执行缓冲区溢出攻击并检测格式化字符串漏洞。)*

此时，`which` 引导的从句被 "was designed" 隔开，它修饰的不再是紧挨着的词，而是隔了老远的 "An automated agent"。

#### 二、如何在长难句中精准揪出被隔开的“先行词”？

在阅读复杂的学术文章或技术文档时，当你看到 `which`, `who`, `that` 时，**千万不要盲目认为它修饰的就是它左边紧挨着的那个名词**。

破解分隔式定语从句，只有两把钥匙：**语法一致** 和 **逻辑一致**。

##### 1. 语法测试（看单复数）

看从句里的谓语动词是单数还是复数，这能帮你直接排除掉一些干扰项。

- **例句：** A new **patch** for these vulnerable *servers* has been released **which addresses** the zero-day exploit.
- **分析：** `which` 紧挨着的是 *servers* (复数)，但从句里的动词是 `addresses` (单数)。因此 `which` 绝对不可能修饰 servers，它跨越了重重阻碍，修饰的是最前面的单数名词 **patch**。

##### 2. 逻辑测试（看意思通不通）

当语法上单复数都一样时，只能把备选名词代入到从句中，看逻辑是否说得通。

- **例句：** The **system** protects the *copyright* of digital media **which uses** perceptual hashing algorithms.
- **分析：** `which` 紧挨着的是 *digital media* 和 *copyright*。
  - 代入测试 1：数字媒体 (digital media) 使用感知哈希算法？（勉强，但不严谨）。
  - 代入测试 2：版权 (copyright) 使用感知哈希算法？（完全不通）。
  - 代入测试 3：系统 (system) 使用感知哈希算法？（完美契合！这是基于哈希比对的数字版权保护系统）。
- 因此，`which` 修饰的是句首的 **system**。

#### 三、最常见的三种“分隔物”

以后在长难句中，如果先行词和关系词之间夹着以下三种成分，一定要立刻警觉：

1. **短谓语动词：** 像 *came, appeared, existed, remained, was found* 等不及物动词或简短被动语态。（例如：A bug *exists* which...）
2. **介词短语：** 先行词自带的介词短语修饰语。（例如：The vulnerability *in the ThinkPHP framework* which...）
3. **时间/地点状语：** （例如：A paper was published *last year* which...）

### （八）混杂式定语从句

它通常指的是最让考生头疼的两大“Boss 级”变体：**带有插入语的复杂定语从句**，以及**双重/多重定语从句**。

它们之所以显得“混杂”，是因为句子成分彼此交织，打破了我们熟悉的“先行词+关系词+完整从句”的线性结构。

#### 变体一：带有插入语的定语从句（“障眼法”结构）

这是各大考试中最爱考的陷阱。它的特点是在关系词和从句真正的谓语之间，硬塞进去一个诸如 *I think, we believe, they say, it seems* 的主谓结构，用来表达说话人的主观态度或猜测。

> **混杂结构：** 先行词 + 关系词 + **(插入语)** + 真正的从句成分

**经典陷阱（Who vs. Whom）：**

- *错误直觉：* The engineer **whom** *I thought* was a hacker turned out to be our admin.

  （很多同学看到 I thought，觉得关系词作了 thought 的宾语，于是选了 whom。）

- *正确句子：* The engineer **who** *I thought* was a hacker turned out to be our admin.

##### 破解技巧：遮挡法（删除法）

当你看到关系词后面紧跟着 *I think / we know / he supposes* 这种表达主观观点的短小主谓结构时，**在脑海里直接把它们划掉**。

把 *I thought* 遮住后，原句变成了：The engineer **who** was a hacker...

很明显，从句里缺的是主语（was 的执行者），所以必须用主格 `who`，绝对不能用宾语 `whom`。

#### 变体二：双重定语从句（“一对多”结构）

正常的定语从句是“一对一”（一个从句修饰一个名词）。而双重定语从句是“一对多”：**两个不同的定语从句，同时修饰同一个先行词**。

> **混杂结构：** 先行词 + **[定语从句 1]** + **[定语从句 2]**

为了不让句子显得臃肿，这种结构有极其严格的“潜规则”：

1. **从句 1 通常很短**，并且经常省略关系代词（that / which）。
2. **从句 2 通常较长**，保留关系词，并且为了避免重复，如果前面默认是 that，后面经常换成 which / who。

**经典例句：**

- Is there anything **(that) you need** **which is not on the list**?
  - *定从 1：* (that) you need （修饰 anything）
  - *定从 2：* which is not on the list （同样修饰 anything）
  - *翻译：* 有没有什么**你需要**但**不在清单上**的东西？

**另一个例子：**

- He is the only man **(that) I know** **who can solve this problem**.

  （他是我认识的唯一一个能解决这个问题的人。）

#### 变体三：嵌套式定语从句（“套娃”结构）

与双重定语从句（并列修饰同一个词）不同，嵌套式是 A 修饰 B，B 又修饰 C，一环套一环。这也是长难句让人喘不过气的元凶。

> **混杂结构：** 先行词 A + 关系词修饰 A + [从句内部出现名词 B] + 关系词修饰 B...

**经典例句：**

- We need to patch the **server** *which hosts the database* **that stores user passwords**.
  - 第一层：*which hosts the database* 修饰 server（运行着数据库的服务器）。
  - 第二层：**that stores user passwords** 修饰 database（存储着用户密码的数据库）。

##### 破解技巧：切断与独立

在阅读或翻译这种“套娃”句时，不要试图用一句中文把它翻出来（“我们需要修补那个运行着存储着用户密码的数据库的服务器”——太别扭了）。

**正确的做法是从关系词处一刀切断，拆成几个短句顺着翻：**

“我们需要给服务器打补丁，这台服务器运行着数据库，而数据库里存着用户密码。”

### （九）多重定语从句

在考研英语（尤其是阅读理解的长难句和翻译题）中，**多重定语从句**是最核心的“拦路虎”。如果说普通的定语从句是给名词加了一个修饰语，那么多重定语从句就是给句子加上了“连环套”。

多重定语从句主要分为两大类：**嵌套式（套娃型）\**和\**并列式（一对多型）**。掌握了这两种结构的拆解方法，长难句就再也难不住你了。

#### 一、 嵌套式定语从句（“套娃”型）

这是最常见的一种复杂结构。它的逻辑是链式的：**A 修饰 B，B 里面又包含一个名词 C，C 后面又跟着一个定语从句修饰它。** 就像套娃一样，一层套一层。

> **结构模型：** 先行词 A + [修饰 A 的定从 (包含名词 B)] + [修饰 B 的定从]

**【考研级实战例句】**

The laboratory is developing an automated **agent** *which can detect the specific format string **vulnerabilities** * **that frequently occur in legacy systems**.

**【硬核拆解】**

1. **第一层：** *which can detect... vulnerabilities* 修饰前面的 **agent**。
2. **第二层：** *that frequently occur in legacy systems* 修饰它紧挨着的前面的名词 **vulnerabilities**（漏洞）。

**【翻译技巧：顺藤摸瓜，步步切断】**

面对这种嵌套结构，**千万不要试图把它翻译成一长串的“……的”**（比如：实验室正在开发一个能检测在老旧系统中经常出现的特定格式化字符串漏洞的自动化代理——这太憋气了）。

正确做法是**从每个关系词处切断，独立成句，代入名词**：

> *翻译：* 实验室正在开发一种自动化代理，**这种代理**能够检测特定的格式化字符串漏洞，**而这些漏洞**在老旧系统中频繁出现。

#### 二、 并列式定语从句（“一对多”型）

与“套娃”不同，并列式的逻辑是**两个或多个定语从句，同时指向前面同一个先行词**。

##### 1. 显性并列（有连词）

两个定语从句之间用 `and`, `but`, `or` 等并列连词连接。关系非常清晰。

- **例句：** We are analyzing the **malware** *which was detected yesterday* **and** *which has infected thousands of servers*.
- **分析：** 两个 `which` 从句同时修饰 malware。
- **翻译：** 我们正在分析那款恶意软件，它不仅是昨天被发现的，而且已经感染了数千台服务器。

##### 2. 隐性并列（无连词，最易出错）

这是考研中最爱设置的陷阱。两个定语从句一前一后直接挨着修饰同一个名词，中间没有 `and`。

它的特征通常是：**先行词 + [短定从（常省略关系词）] + [长定从（有关系词）]**。

- **例句：** Is there **anything** *(that) you need for the penetration test* **which is not covered in this manual**?
- **分析：**
  - 第一个从句：*(that) you need for the penetration test* 修饰 anything。
  - 第二个从句：*which is not covered in this manual* 跨越了第一个从句，依然修饰句首的 anything。
- **翻译融合：** 有没有什么你做渗透测试需要、但这本手册里没有涵盖的**东西**？

#### 三、 考研实战：多重定从的终极破解三步法

当你拿到一个极其复杂的长难句，发现里面有多个 `who, which, that` 时：

1. **抓主干，剥洋葱：** 先别管从句，找出整句话的“主谓宾”。把由关系词引导的从句先用括号 `[ ]` 括起来。
2. **找归属，画箭头：** 看看括号里的从句到底修饰谁。如果紧挨着名词，可能是套娃；如果从句里缺成分且逻辑指向更前面的核心词，可能是隐性并列。
3. **切成短句，补充代词（翻译高分秘诀）：** 不要勉强拼凑成一句中文，把从句切下来，用“这”、“他”、“该系统”等代词衔接。

> **我们来试一道地狱难度的考研长难句：**
>
> *He is the only security expert I know who can trace the hacker that launched the attack which paralyzed the entire network.*
>
> - **试着切开翻：** 他是我认识的唯一一位安全专家，他能追踪到那个黑客，而那个黑客发起了瘫痪整个网络的攻击。

### （十）省略式定语从句

**省略式定语从句**是英语母语者最爱用的表达方式，也是考研英语长难句中用来“隐藏句子结构、增加阅读难度”的常见手段。

在定语从句中，并不是所有的关系词都可以省略。省略的核心原则是：**不能影响句子的基本结构和意思的表达。**

英语中定语从句的省略主要分为三大类，其中最核心的是前两类：

#### 一、 关系代词作“宾语”时的省略（最常见）

当关系代词（`that`, `which`, `whom`）在定语从句中充当**动词的宾语**或**介词的宾语**时，可以直接省略。

**1. 作动词宾语时的省略**

- **完整：** The exploit code **(that/which)** he wrote yesterday is very effective.
- **分析：** 从句 *he wrote yesterday* 中，wrote 是及物动词，缺宾语，`that/which` 就是 wrote 的宾语，所以可以省略。
- **省略后：** The exploit code **he wrote yesterday** is very effective. （他昨天写的利用代码非常有效。）

**2. 作介词宾语时的省略（注意介词位置）**

- **完整（介词在后）：** This is the target server **(which)** we are connecting **to**.
- **省略后：** This is the target server **we are connecting to**.
- ⚠️ **绝对禁忌：** 如果介词被提前了，关系代词**绝对不能**省略！
  - *错：* This is the target server to we are connecting.
  - *对：* This is the target server **to which** we are connecting.

> **长难句破解技巧：** 在阅读中，一旦你看到 **“名词1 + 名词2/代词 + 动词”** 紧紧挨在一起（例如 `the algorithm we use`），中间没有任何标点或连词，这 100% 是一个省略了宾语关系代词的定语从句！从名词2开始加个括号划断即可。

#### 二、 “关系词 + be 动词”的打包省略（定从简化）

这其实是英语中高级语法的“非谓语动词作定语”和“介词短语作定语”。当定语从句的结构是 **“关系代词作主语 + be动词 + 其他成分”** 时，我们可以把“关系代词 + be”一起打包扔掉。

省略后，剩下的成分会直接贴在名词后面作后置定语。

**1. 剩下现在分词（V-ing） $\rightarrow$ 表示主动或进行**

- **完整：** The agent **which is scanning** the network will report the results soon.

- **打包省略：** The agent **scanning** the network will report the results soon.

  *(正在扫描网络的代理很快会报告结果。)*

**2. 剩下过去分词（V-ed） $\rightarrow$ 表示被动或完成**

- **完整：** The zero-day vulnerability **which was discovered** last week has been patched.

- **打包省略：** The zero-day vulnerability **discovered** last week has been patched.

  *(上周被发现的零日漏洞已被修复。)*

**3. 剩下形容词或介词短语**

- **完整：** The administrator **who is responsible for this server** is on leave.

- **打包省略：** The administrator **responsible for this server** is on leave.

  *(负责这台服务器的管理员休假了。)*

#### 三、 关系副词的省略

正如我们之前在讨论 `that` 的用法时提到的，当先行词是一些表示泛指的时间、地点、原因、方式的名词时，引导从句的关系副词可以省略。

- **the reason (why)：** This is the reason **(why)** the program crashed.
- **the time (when)：** Do you remember the time **(when)** we configured the firewall?
- **the way (in which / that)：** I don't like the way **(that / in which)** the data is encrypted. *(注意：the way 后面的关系词最常被省略。)*
