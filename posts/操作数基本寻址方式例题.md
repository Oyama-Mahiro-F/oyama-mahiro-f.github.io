## 完整题目

某计算机字长为16位，主存地址空间大小为128 KB，**按字编址**。采用单字长指令格式，指令各字段定义如下：

| 15 ~ 12 | 11 ~ 9 | 8 ~ 6  | 5 ~ 3  | 2 ~ 0  |
| :-----: | :----: | :----: | :----: | :----: |
| **OP**  | **Ms** | **Rs** | **Md** | **Rd** |

- **OP**：操作码；**Ms/Md**：源/目的操作数的寻址方式；**Rs/Rd**：源/目的操作数的寄存器编号。
- 寻址方式定义如下表：

| 寻址方式编码 | 助记符           | 含义  |
| :----------: | :--------------- | :---- |
|     000B     | 寄存器直接       | Rn    |
|     001B     | 寄存器间接       | (Rn)  |
|     010B     | 寄存器间接、自增 | (Rn)+ |
|     011B     | 相对             | D(Rn) |

请回答下列问题：

**（1）** 该指令系统最多可有多少条指令？该计算机最多有多少个通用寄存器？存储器地址寄存器（MAR）和存储器数据寄存器（MDR）至少各需要多少位？  
**（2）** 转移指令的目标地址范围是多少？  
**（3）** 若操作码 `0010B` 表示加法操作（助记符为 `add`），寄存器 `R4` 和 `R5` 的编号分别为 `100B` 和 `101B`。已知 `R4` 的内容为 `1234H`，`R5` 的内容为 `5678H`，地址 `1234H` 中的内容为 `5678H`，地址 `5678H` 中的内容为 `1234H`。则汇编语句 `add (R4), (R5)+`（逗号前为源操作数，逗号后为目的操作数）对应的**机器码是什么**（用十六进制表示）？该指令执行后，哪些寄存器和存储单元的内容会改变？改变后的内容是什么？

---

## 标准答案与详细解析

### 第（1）问解析
- **最多指令条数**：OP 字段占 4 位 → \( 2^4 = 16 \) 条。
- **最多通用寄存器数**：Rs / Rd 各占 3 位 → \( 2^3 = 8 \) 个。
- **MAR 位数**：主存 128KB = \( 2^{17} \) 字节，按字编址（字长16位=2字节）→ 可寻址单元数 = \( 2^{17} / 2 = 2^{16} \) → MAR 至少 **16 位**。
- **MDR 位数**：CPU 与主存一次交换的数据量 = 1 个字 = 16 位 → MDR 至少 **16 位**。

> **答案**：16条；8个；16位；16位。

---

### 第（2）问解析
- 相对寻址公式：`目标地址 = (PC) + (Rn)`。
- 该机字长16位 → PC 和 Rn 均为 **16 位寄存器**。
- 两个 16 位数相加，结果保留低 16 位（高位自然溢出截断）。
- 16 位无符号整数的取值范围为 \( 0 \) 到 \( 2^{16}-1 \)，即十六进制 **0000H ~ FFFFH**。

> **答案**：0000H ~ FFFFH。

---

### 第（3）问详细拆解（重点）

**第一步：写出机器码**

根据指令格式 `| OP(4) | Ms(3) | Rs(3) | Md(3) | Rd(3) |`：

- `add` → OP = `0010B`
- 源操作数 `(R4)` → 寻址方式为“寄存器间接”（`001B`），寄存器 R4 编号 `100B` → Ms=`001`, Rs=`100`
- 目的操作数 `(R5)+` → 寻址方式为“寄存器间接、自增”（`010B`），寄存器 R5 编号 `101B` → Md=`010`, Rd=`101`

拼接：`0010 001 100 010 101B` = `0010 0011 0001 0101B` = **`2315H`**

> **机器码答案**：**`2315H`**

---

**第二步：指令执行过程（按寻址规则）**

| 执行步骤               | 操作详情                                                     | 计算结果                       |
| :--------------------- | :----------------------------------------------------------- | :----------------------------- |
| ① 取源操作数           | 源为 `(R4)` → 有效地址 = R4 = `1234H`<br>取出内容 = 内存 `[1234H]` = `5678H` | 源操作数 = **`5678H`**         |
| ② 取目的操作数（旧值） | 目的为 `(R5)+` → 有效地址 = R5 = `5678H`<br>取出内容 = 内存 `[5678H]` = `1234H`<br>**（先取数，后自增）** | 目的旧值 = **`1234H`**         |
| ③ 自增副作用（修改R5） | R5 的内容按字编址自增 1                                      | **R5 = `5678H + 1` = `5679H`** |
| ④ 执行加法运算         | 目的新值 = 目的旧值 + 源操作数<br>= `1234H + 5678H` = `68ACH` | 运算结果 = **`68ACH`**         |
| ⑤ 写回结果             | 将结果写入目的有效地址（即步骤②中的地址 `5678H`）            | **内存 `[5678H]` = `68ACH`**   |

---

**第三步：最终变化的寄存器和存储单元**

| 变化对象             | 变化前内容 | 变化后内容          | 变化原因                     |
| :------------------- | :--------- | :------------------ | :--------------------------- |
| **存储单元 `5678H`** | `1234H`    | **`68ACH`**         | 加法结果写回                 |
| **通用寄存器 R5**    | `5678H`    | **`5679H`**         | 自增寻址副作用               |
| **通用寄存器 R4**    | `1234H`    | 保持不变（`1234H`） | 仅作为地址指针，未被修改     |
| **存储单元 `1234H`** | `5678H`    | 保持不变（`5678H`） | 仅作为源操作数读取，未被修改 |

> **补充（非必须但常考）**：该加法指令还会影响**程序状态字（PSW/标志寄存器）**。由于 `1234H + 5678H = 68ACH`，没有产生进位（CF=0），结果不为零（ZF=0），亦无符号溢出（OF=0）。若题目要求“所有改变的内容”，通常需提一句“标志寄存器相应更新”，但考研核心得分点仍是上表。
